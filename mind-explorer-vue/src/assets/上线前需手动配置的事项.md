## 一、Netlify 环境变量（必做区）

位置：**Netlify 后台 → 你的站点 → Deploys → Environment variables**（或 Site settings → Environment variables）。
新增后需重新部署（Deploys → Trigger deploy）才能生效。

> 注：实际部署的函数目录为 `netlify/functions/`（已在 `netlify.toml` 中配置 `functions = "netlify/functions"`）。顶层 `netlify/*.js` 是早期遗留副本，**不会**被部署，可忽略。

| # | 变量名 | 是否必填 | 代码默认值 | 不配的后果 | 去哪拿 |
|---|--------|----------|------------|------------|--------|
| 1 | `SUPABASE_SERVICE_ROLE_KEY` | **必填** | 空（无） | 服务端记忆、小木多轮上下文、心情日记、收藏、打卡、治愈瞬间投稿 **全部降级**（函数返回「未启用存储」） | Supabase 后台 → Project Settings → API → `service_role` 密钥（**仅服务端用，切勿泄露到前端**） |
| 2 | `GITHUB_CLIENT_ID` | **必填（若启用 GitHub 登录/轻账号）** | 无 | 「GitHub 登录」点了会跳到 `client_id=undefined` 的授权页，登录失败 | 注册 GitHub OAuth App 后获得（见第三节） |
| 3 | `GITHUB_CLIENT_SECRET` | **必填（同上）** | 无 | OAuth 回调换取 token 失败，登录不可用 | 同上，OAuth App 的 secret |
| 4 | `GITHUB_REPO_TOKEN` | **必填（若启用社区/积分/认同/评论/复习卡互动/旧投稿）** | 空（无） | 社区列表、贡献榜、认同/评论、复习卡片互动等 GitHub Issues 支撑功能读写为空或报错 | 生成一个有 `repo` 权限的 GitHub Personal Access Token（PAT） |
| 5 | `ADMIN_PWD` | **建议改** | `mind2024` | 不配则审核后台 `/admin` 用默认弱密码，存在被他人进入的风险 | 改成你自己的强密码，访问 `/admin` 时输入 |
| 6 | `DEEPSEEK_API_KEY` | 可选 | 空（无） | 小木对话、每日陪伴语、每日文案、图片配文 **走内置模板**（仍可用，只是不够个性化） | DeepSeek 平台 → API Keys |
| 7 | `DEEPSEEK_BASE_URL` | 可选 | `https://api.deepseek.com/v1` | 一般无需改；仅当你用兼容代理/自建网关时改 | —— |
| 8 | `SUPABASE_URL` | 可选 | `https://bbdfeiceezcbcbsbnznr.supabase.co`（代码内置） | 一般无需配；仅当 Supabase 项目地址变化时改 | Supabase 项目 URL |
| 9 | `SUPABASE_ANON_KEY` | 可选 | 代码内置硬编码的 publishable key | sunny-push 在没有 service_role 时的回退；一般无需配 | Supabase `anon` 公钥 |
| 10 | `GITHUB_REPO_OWNER` | 可选 | `kangkang-del` | 仅当代码仓库不在该账户下时才改（否则社区/积分读写错仓库） | 你的 GitHub 用户名 |
| 11 | `GITHUB_REPO_NAME` | 可选 | `mind-explorer` | 同上 | 你的仓库名 |
| 12 | `DRY_RUN` | 调试用 | 不设 | sunny-push 调试开关，正式环境无需配 | —— |

> `URL` 变量是 **Netlify 自动注入** 的（部署站点地址），OAuth 回调用，无需手动设置；本地调试时才需要关心。

## 二、GitHub Secrets（必做区，用于每日定时推送）

位置：**GitHub 仓库 → Settings → Secrets and variables → Actions → New repository secret**。

| 变量名 | 是否必填 | 不配的后果 | 说明 |
|--------|----------|------------|------|
| `SITE_URL` | 可选但建议 | 每日定时推送任务跳过（**非阻断**：前端「今日精选 / 心灵晴天」打开时会兜底懒触发，所以没配也能跑，只是无人访问时不会有预置内容） | 填部署后的站点地址，**不含末尾斜杠**，例如 `https://mind-explorer.netlify.app` |

> 配套文件：`.github/workflows/sunny-push.yml`（cron `0 0 * * *` UTC = 北京时间 08:00）。该 Actions 仅在仓库 **默认分支** 上运行，需确保仓库已启用 GitHub Actions。

## 三、第三方平台手动操作（非环境变量）

### 3.1 注册 GitHub OAuth App（启用「轻账号 / GitHub 登录」必做）
1. 打开 **github.com → 右上角头像 → Settings → Developer settings → OAuth Apps → New OAuth App**。
2. **Application name**：心灵探索（或任意）。
3. **Homepage URL**：你的部署站点地址，例如 `https://mind-explorer.netlify.app`。
4. **Authorization callback URL**：`https://<你的站点>/.netlify/functions/auth-callback`（必须精确匹配，否则回调 404）。
5. 创建后得到 `Client ID` 与 `Client secret`，分别填入 Netlify 的 `GITHUB_CLIENT_ID` / `GITHUB_CLIENT_SECRET`。
6. 生成的 `GITHUB_REPO_TOKEN`（见上表第 4 项）建议给 `repo`  scope，供社区/积分等功能读写 Issues。

> 若你 **不打算** 提供 GitHub 登录，可跳过 3.1，并把前端登录入口视为「访客（guest）模式」——访客模式不依赖任何密钥即可使用基础功能。

### 3.2 Supabase 数据库表（✅ 已通过 MCP 迁移完成，你无需建表）
以下表与 RLS 策略已执行 SQL 迁移（`sql/*.sql`），**RLS 启用但无开放 policy，所有读写经 Netlify Function + service_role 中转**：
- `companion_messages`（小木对话记忆）
- `user_profiles`（轻量用户画像）
- `mood_diary`（心情日记）
- `user_cards`（治愈瞬间投稿）
- `user_favorites`（收藏）
- `user_checkins`（打卡）

> 你唯一要做的就是把 **3.1 之外** 的 Supabase `service_role` 密钥（`SUPABASE_SERVICE_ROLE_KEY`）填进 Netlify 环境变量（见上表第 1 项）。
> 社区帖子 / 积分 / 认同评论等走 **GitHub Issues** 通道，由 `GITHUB_REPO_TOKEN` 支撑，与 Supabase 无关。

## 四、按功能模块对照：哪些没配会怎样

| 功能模块 | 关键依赖 | 未配置时的表现 |
|----------|----------|----------------|
| 小木对话（多轮 + 记忆） | `SUPABASE_SERVICE_ROLE_KEY` + `DEEPSEEK_API_KEY` | 无 service_role → 回退浏览器本地历史（无跨设备记忆）；无 DeepSeek key → 模板语录（仍流式） |
| 每日主动陪伴语 | `SUPABASE_SERVICE_ROLE_KEY` + `DEEPSEEK_API_KEY` | 无 service_role → 通用暖语；无 key → 按心情趋势模板 |
| 心情日记 + 情绪曲线 | `SUPABASE_SERVICE_ROLE_KEY` | 无 → 保存/读取返回「未启用存储」 |
| 收藏（四类） | `SUPABASE_SERVICE_ROLE_KEY` | 无 → 收藏按钮无效 |
| 打卡 / 连续天数 | `SUPABASE_SERVICE_ROLE_KEY` | 无 → 打卡静默跳过 |
| 治愈瞬间投稿 + 审核 | `SUPABASE_SERVICE_ROLE_KEY` + `ADMIN_PWD` | 无 service_role → 投稿失败；默认密码 `mind2024`（建议改） |
| 社区交流 / 认同 / 评论 | `GITHUB_REPO_TOKEN` | 无 token → 列表可能为空或仅极少量公开数据 |
| 贡献榜 / 积分 | `GITHUB_REPO_TOKEN` | 无 token → 拉取失败 |
| GitHub 登录 / 轻账号 | `GITHUB_CLIENT_ID` + `GITHUB_CLIENT_SECRET` + OAuth App | 未注册/未填 → 登录失败；可用 guest 模式替代 |
| 每日自动推送（心灵晴天） | `SITE_URL`（GitHub Secret） | 未配 → 定时任务跳过，前端兜底触发仍可用 |
| 自助工具箱 / 危机文案 | 无外部依赖 | 纯前端，始终可用 |

## 六、最小可上线清单（不折腾版）

只想要网站先跑起来？按这个顺序配 **3 项** 即可：

1. Netlify `SUPABASE_SERVICE_ROLE_KEY` ← 让记忆/心情/收藏/打卡/投稿可用
2. Netlify `ADMIN_PWD` ← 改成自己的密码（不然审核后台用默认弱密码）
3. （可选）GitHub Secret `SITE_URL` ← 开启每日自动推送

其余（`DEEPSEEK_API_KEY`、`GITHUB_CLIENT_ID/SECRET/TOKEN`）均为「配了更好、不配也能用（降级）」的可选增强。
